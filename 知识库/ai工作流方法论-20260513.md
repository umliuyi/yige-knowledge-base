# AI工作流制作方法论
> 来源：一哥口述整理（2026-05-13）

## 核心定义
AI工作流 = 把一件事拆成固定步骤，让AI+工具自动跑完。
典型结构：`输入 → AI处理 → 工具调用 → 再AI处理 → 输出`

## 五步法

### 第一步：明确目标与边界
写清楚：
- 要解决什么问题？
- 输入是什么？
- 输出要什么格式？
- 要不要联网/查数据/用内部数据？

### 第二步：拆解成节点
每个节点 = 可独立执行、可串联的步骤。
示例（会议纪要工作流）：
1. 输入：上传录音
2. 转写：AI语音转文字
3. 提炼：AI提取结论、决策、待办
4. 格式化：AI生成标准模板
5. 输出：保存到文档 + 推送群里

### 第三步：选工具平台
- 零代码：Coze（字节）、Dify、N8N、飞书智能流程
- 进价开源：N8N（节点多，支持Docker本地部署）
- 大模型专用：Flowise（拖拽LLM、RAG、工具节点）
- 代码可控：LangChain、LlamaIndex

### 第四步：可视化编排
拖拽节点 → 连线 → 配参数（模型、提示词、温度）

提示词结构化：
`角色 + 任务 + 格式 + 约束 + 例子`

### 第五步：测试→调优→固化
1. 小批量测试
2. 优化提示词
3. 保存为模板

## 龙虾团队工作流原则
- 每个工作流必须先回答：输入是什么、输出是什么
- 每步有检查点，不是黑盒
- 养成先用工作流思考的习惯，不是拍脑袋
