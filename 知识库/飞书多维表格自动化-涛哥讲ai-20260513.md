## 核心框架

飞书多维表格自动化分为两种：

| 类型 | 适用场景 | 复杂度 |
|------|---------|--------|
| 普通自动化 | 简单任务（新增→通知） | 低 |
| 工作流 | 复杂任务（多步骤判断） | 高 |

---

## 自动化组成

### 触发条件

| 类型 | 说明 |
|------|------|
| 数据变更 | 记录新增/修改/删除时触发 |
| 时间驱动 | 定时触发（每天/每周） |
| 交互触发 | 按钮/表单提交 |
| 外部类 | Webhook/API触发 |

### 执行操作

| 类型 | 说明 |
|------|------|
| 基础操作 | 发送消息/更新字段 |
| 数据操作 | 查找/新增/删除记录 |
| 飞书协作 | @人员/创建日程/发送通知 |
| 逻辑控制 | 条件分支/循环 |

## 具体示例

### 示例1：任务管理自动化

```
触发：新任务记录新增
↓
条件：任务状态="待处理"
↓
执行：发送通知给负责人
↓
执行：更新任务状态为"已通知"
```

### 示例2：库存管理自动化

```
触发：物品申领表单提交
↓
条件：库存 > 申领数量
↓
执行：通过申请
执行：发送确认消息
↓
执行：更新库存数量
↓
否则：
执行：拒绝申请
执行：发送缺货通知
```

### 示例3：客户跟进自动化

```
触发：客户状态变更
↓
条件：状态="待跟进"
↓
执行：发送跟进提醒给业务员
↓
执行：创建日程（2天后跟进）
```

## AI Agent节点（重要！）

涛哥强调的杀手级功能：

```
在自动化/工作流中加入"AI处理节点"
↓
AI自动处理：文本改写/总结/分类
↓
结果写入表格字段
```

**应用场景：**
- 客户反馈 → AI自动分类（好评/投诉/建议）
- 销售记录 → AI提取关键信息
- 内容素材 → AI生成摘要
